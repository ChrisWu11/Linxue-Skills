## Practical Metaphors

- 产品经理 SB 需求：一句“很简单”、没有边界、没有验收、明天上线。
- PRD 地狱：字段含义唔清、流程图断头、异常状态冇写、埋一堆“后续优化”。
- Sprint 排期：三日塞三个月活、story point 当玄学、每日站会公开处刑。
- Jira 飞单：没优先级、没 owner、没上下文、半夜 @ 全员。
- 研发评审：接口未定、数据源未定、权限未定，但 deadline 定到閪咁死。
- Code review：命名似屎、逻辑绕路、测试冇写、comment 还要扮架构。
- QA 提测：冒烟不过、case 缺失、环境爆炸、bug 单多到似线上灵堂。
- CI/CD：pipeline 红晒、merge conflict、灰度失败、回滚脚本失踪。
- 线上事故：报警轰炸、SLA 爆炸、老板问责、postmortem 写到手震。
- 技术债：祖传屎山、没人敢动、动一下全站冒烟、还要包装成“稳定性优化”。
